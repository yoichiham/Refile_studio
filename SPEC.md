# SPEC.md — Refile.studio 仕様書（v1.1）

- 文書名：仕様書
- 対象：ブラウザだけで完結するファイル変換ユーティリティ集（テキスト/画像/PDF/Markdown）。サーバー送信なし。
- 作成日：2026-06-24
- v1.1：画像→PDF / PDF結合 / PDF分割・ページ操作 を追加。PWA化・全ツールD&D 対応。

---

## 1. 目的・背景

- AIの返答テキストや手元の画像/PDFを、外部にアップロードせずローカルだけで素早く変換したい。個人・友人での日常利用を想定。
- 変換系の小ツールは個別に探すと散らばる。1か所（.studio）に集約し、ツールを後から増やせる土台を作る。
- プライバシー重視：ファイルやテキストを一切サーバーに送らず、すべてブラウザ内（クライアントサイド）で処理する。

### 1.1 スコープ

**対象（v1 に含む）**
- ① テキスト → `.md` ダウンロード
- ② 画像リサイズ＆フォーマット変換（PNG/JPEG/WebP 出力）
- ③ PDF → 画像（JPEG、複数ページは ZIP）
- ④ Markdown → PDF（ベクターPDF、Noto Sans JP 埋め込み）
- ⑤ 画像 → PDF（複数画像を1つのPDFに結合）(v1.1)
- ⑥ PDF 結合 (v1.1)
- ⑦ PDF 分割・ページ操作（並べ替え/削除/抽出）(v1.1)
- 各ツールをサイドバー＋ハッシュルーティングで切替。ツールはレジストリで管理し追加容易にする
- 全ツール共通のドラッグ&ドロップ対応、PWA化（オフライン利用）(v1.1)

**対象外（v1 で作らない）**
- サーバー/バックエンド、ユーザー認証、クラウド保存、共有リンク
- 入力データの永続化（履歴・自動保存はしない）
- 多言語UI（日本語のみ）
- ブラウザ印刷ダイアログ方式のPDF出力（自動ダウンロード・自動命名要件と相反するため不採用）
- PDF以外 → PDF（Word等）、動画/音声の変換
- アニメーションGIFの全フレーム保持（1フレーム目のみ静止画化）
- モバイル最適化の作り込み（PC前提。レスポンシブは最低限）
- ②の複数画像一括変換、ライブプレビュー、ダークモード（将来検討）

## 2. 全体アーキテクチャ

- React + TypeScript + Vite の SPA。状態は各ツール内のローカルstate中心、永続化なし。
- ルーティングは react-router の **HashRouter**（`#/md-file` 等）。GitHub Pages の静的配信でも直接アクセス・リロード・ブックマークが壊れない。
- ツールは「ツールレジストリ（配列）」で定義。サイドバーとルートは登録から自動生成。新ツールは1ファイル＋1登録で追加。
- 処理ライブラリ：
  - 画像：Canvas API（drawImage + canvas.toBlob）
  - PDF→画像：pdfjs-dist（Web Worker）＋ JSZip
  - Markdown解析：markdown-it / marked で AST を得て pdfmake のドキュメント定義へ変換
  - Markdown→PDF：pdfmake（Noto Sans JP を vfs に動的登録）
  - PDF操作（結合/分割/ページ操作/画像埋め込み）：pdf-lib。役割分担＝pdfjs-dist は描画、pdf-lib は編集。
- デプロイ：GitHub Pages（静的）。Vite の `base` を `'/<リポジトリ名>/'` に設定。
- PWA：vite-plugin-pwa で manifest＋Service Worker を生成し、アセットをキャッシュしてオフライン動作。SW/manifest の scope を base path（`'/<リポジトリ名>/'`）に整合させる。

### 2.1 不変条件

1. 入力（テキスト・画像・PDF）を一切ネットワーク送信しない。すべてブラウザ内で処理する。
2. 入力データを永続化しない（localStorage等に保存しない）。リロードで消えてよい。
3. GitHub Pages の base path と HashRouter を前提に、絶対パス直書きをしない。
4. ツール追加はレジストリ経由のみ。ルーティング/サイドバーを個別にハードコードしない。

## 6. 機能仕様

### 6.1 テキスト → .md ダウンロード（id: md-file）
- テキストエリアの内容を `.md` として Blob 生成 → ダウンロード。
- バリデーション：空欄 → 処理中断、「テキストを入力してください」を画面表示。0文字はエラー。
- 上限目安：10万文字まで正常処理。
- ファイル名：`YYYYMMDD_HHMMSS.md`（ローカル時刻で一意付与）。
- クライアントのみで完結（サーバー送信なし）。

### 6.2 画像リサイズ＆フォーマット変換（id: image-convert）
- 受付形式：PNG / JPEG / WebP / GIF / BMP、最大10MB。
- 指定：幅・高さ（px）、出力フォーマット（PNG / JPEG / WebP）。
- アスペクト比：デフォルト維持。片方入力でもう片方を自動計算し入力欄に反映。
- 出力：Canvas で描画 → 指定フォーマットで toBlob → ダウンロード。
- GIF 等アニメは最初の1フレームのみ静止画化。
- エラー：
  - 対象外形式（PDF/テキスト等）→「対応していないファイル形式です」
  - 幅 or 高さが 0 以下 or 10000 以上 →「正しいサイズを指定してください」
  - 10MB 超 →「ファイルサイズは10MB以下にしてください」
- 境界値：1px 正常、10.0MB 丁度は正常、10.01MB はエラー。

### 6.3 PDF → 画像（id: pdf-to-image）
- pdfjs-dist で各ページを canvas にレンダ → JPEG 化。
- 1ページ：JPEG 1枚をダウンロード。複数ページ：全ページ JPEG 化し ZIP で一括ダウンロード（JSZip）。
- 品質選択：標準（150dpi / JPEG 80%、デフォルト）／高画質（300dpi / JPEG 90%）。出力フォーマットは JPEG 固定。
- 処理中はロード画面を表示。100ページまで正常に全ページ変換。
- エラー：
  - パスワード保護PDF →「パスワードで保護されたPDFは変換できません」
  - 破損PDF →「ファイルを読み込めませんでした」

### 6.4 Markdown → PDF（id: md-to-pdf）
- テキストエリアの Markdown を解析 → pdfmake でベクターPDF生成 → ダウンロード。
- 見出し / リスト / 表 / コードブロックを装飾。**テキスト選択・検索可能**。
- 日本語：Noto Sans JP を埋め込み（vfs へ動的登録）し文字化けを防止。
- 用紙：A4縦、余白 上下左右 20mm。長文・大きな表は自動改ページ。
- デザイン：白黒基調のシンプル。見出しはサイズ/太さ（必要なら控えめな下線）、表・コードブロックは薄いグレーの枠線/背景。過度な装飾なし。
- バリデーション：空欄 → 処理中断「テキストを入力してください」。
- ファイル名：`YYYYMMDD_HHMMSS.pdf`。

### 6.5 画像 → PDF（id: image-to-pdf, v1.1）
- 複数画像（PNG/JPEG/WebP/GIF/BMP）をアップロードし、並び順に1つのPDFへ結合（1画像 = 1ページ）。
- レイアウト：A4縦、余白20mm、アスペクト比維持でページ内にフィット。
- 実装：pdf-lib（embedPng/embedJpg）。WebP/GIF/BMP は Canvas 経由で PNG/JPEG に正規化してから埋め込む（§6.2 の Canvas 基盤を流用。GIF は1フレーム目）。
- 出力：`YYYYMMDD_HHMMSS.pdf`。
- エラー：画像0枚 →「画像を選択してください」。非対応形式・10MB超は §6.2 と同じ文言。

### 6.6 PDF 結合（id: pdf-merge, v1.1）
- 複数PDFを並び順に結合し1つのPDFに。実装：pdf-lib。
- 出力：`YYYYMMDD_HHMMSS.pdf`。
- エラー：
  - 1ファイル以下 →「PDFを2つ以上選択してください」
  - パスワード保護PDF →「パスワードで保護されたPDFは操作できません」
  - 破損PDF →「ファイルを読み込めませんでした」

### 6.7 PDF 分割・ページ操作（id: pdf-pages, v1.1）
- 1つのPDFを読み込み、ページのサムネイル一覧を表示。ページの並べ替え・削除・抽出（範囲/選択）が可能。
- 出力：編集後の単一PDF。抽出を複数グループで行う場合は ZIP。
- 実装：pdfjs-dist でサムネイル描画（§6.3 の基盤流用）＋ pdf-lib でページのコピー/削除/並べ替え。
- エラー：パスワード保護 →「パスワードで保護されたPDFは操作できません」、破損 →「ファイルを読み込めませんでした」。

## 7. UI/UX 方針

- レイアウト：左サイドバー（ツール一覧、レジストリから生成）＋右メインにアクティブツール。
- ルーティング：HashRouter。`#/` はトップ（ツール一覧/説明）、`#/<tool-id>` で各ツール。未知IDはトップへ。
- ロケール：日本語のみ。
- デザイン：白基調のシンプルなUI。エラーは各ツール内にインライン表示。
- レスポンシブ：PC前提。狭幅ではサイドバー折りたたみ程度の最低限対応。
- 重い処理（PDF変換等）はローディング表示でブロック。
- ファイル入力は全ツール共通でドラッグ&ドロップに対応（クリック選択も維持）。共通フック/コンポーネントを `src/lib` に置き各ツールで利用。

## 8. 非機能要件

- 対応ブラウザ：モダンevergreen（Chrome / Edge / Firefox / Safari 最新）。
- 性能：画像10MB・PDF100ページを現実的時間で処理。重い処理は pdfjs の worker を活用し、ページ逐次処理＆canvas解放でメモリを抑える。
- セキュリティ/プライバシー：サーバー送信なし。外部解析・トラッキングなし。
- コスト：静的ホスティングのみ（実質ゼロ）。
- バンドル：Noto Sans JP の全グリフ埋め込みはサイズ過大 → フォントは動的ロードし vfs 登録（初回PDF生成時に fetch）。
- PWA：オフライン利用可能（Service Worker キャッシュ）。初回ロード後はネット不要。更新は autoUpdate。

## 9. 主要トレードオフ・既知の制約

1. **MD→PDF にベクター（pdfmake）採用**：テキスト選択可・軽量・改ページ綺麗。代償＝Markdown→pdfmake定義への変換実装が必要＋日本語フォントの取り回し。緩和＝AST から限定要素（見出し/段落/リスト/表/コード/強調）だけマッピングし、未対応記法はプレーン表示にフォールバック。フォントは動的ロードでバンドル肥大を回避。
2. **HashRouter 採用**：URLに `#` が入る代償。緩和＝GitHub Pages で 404 や base path 事故を確実に回避でき、個人用途では `#` 許容。
3. **クライアント完結の PDF→画像**：大容量PDFはブラウザのメモリ/CPU依存。緩和＝ページ逐次処理＆canvas解放、ロード表示、想定上限100ページを明記。
4. **GIF アニメは1フレームのみ**：代償＝アニメ保持不可。緩和＝仕様として明示。
5. **永続化なし**：代償＝リロードで入力消失。緩和＝意図的（プライバシー優先）であることを明示。
6. **pdf-lib と pdfjs-dist の併用**：PDFライブラリ2本でバンドル増の代償。緩和＝ツール単位の動的 import で遅延ロード。
7. **PWA キャッシュ**：古いアセットを配信する恐れ。緩和＝autoUpdate ＋ base path 整合で SW を入れ替え。

## 10. 受入条件（抜粋）

- テキスト入力ありで「ダウンロード」→ `YYYYMMDD_HHMMSS.md` がDLされ内容一致。空欄→「テキストを入力してください」表示、DLされない。
- 画像で幅のみ入力→高さが比率維持で自動補完。PNG/JPEG/WebP 各出力でDLできる。非対応形式・サイズ範囲外・10MB超で規定エラー文言。10.0MBは成功、10.01MBはエラー。
- 1ページPDF→JPEG1枚DL。複数ページ→ZIPでDL。標準/高画質を切替できる。パスワード保護・破損で規定エラー文言。100ページ完走。
- Markdown で PDF 生成→見出し/表/コードが装飾され、日本語が文字化けせず、PDF内テキストを選択/検索でき、A4縦・余白20mmで自動改ページされる。空欄→規定エラー。
- GitHub Pages にデプロイ後、`#/<tool-id>` 直アクセス・リロードで各ツールが表示される（404にならない）。
- どの操作でもネットワークにファイル/テキストが送信されない（DevToolsで送信なしを確認）。
- 複数画像で画像→PDF→ページ数=画像数のPDFがDLされ、各画像が比率維持で配置される。0枚→規定エラー。
- 2つ以上のPDFを結合→ページが順に連結された単一PDFがDL。1つ以下→規定エラー。
- PDFのページを削除/並べ替え/抽出→指定通りのPDF（複数抽出はZIP）がDL。
- 全ファイル系ツールでドラッグ&ドロップでファイル投入できる。
- オフライン（ネット切断）でも初回ロード済みなら各ツールが動作する。
